# DevOps AWS Pipeline

End-to-end **Infrastructure as Code (IaC)** and **CI/CD** pipeline that provisions Ubuntu on AWS with **Terraform**, deploys a production-style web experience with **Ansible**, and orchestrates the full workflow through **Jenkins**.

## Overview

This repository automates cloud infrastructure and application delivery:

| Layer | Technology | Responsibility |
|-------|------------|----------------|
| **CI/CD** | Jenkins | Checkout → Terraform → Ansible → HTTP health check |
| **IaC** | Terraform | Ubuntu EC2, security groups, dynamic Ansible inventory |
| **Config Mgmt** | Ansible | Nginx + futuristic static web UI on the target host |
| **Cloud** | AWS | Compute, networking, key-pair SSH access |

After a successful pipeline run, the deployed site is available at:

```text
http://<EC2_PUBLIC_IP>/
```

## Architecture

```text
┌─────────────┐     ┌──────────────────┐     ┌─────────────────┐
│   Jenkins   │────▶│    Terraform     │────▶│  Ubuntu EC2     │
│  Pipeline   │     │  (AWS Provider)  │     │  (us-east-1)    │
└──────┬──────┘     └────────┬─────────┘     └────────▲────────┘
       │                     │                         │
       │              inventory.ini                    │
       │              (auto-generated)                 │
       └─────────────────────┴──── Ansible SSH ───────┘
```

### Pipeline stages

1. **Checkout** — Clone repository from SCM  
2. **Terraform Plan & Apply** — Provision EC2 and write `ansible/inventory/inventory.ini`  
3. **Export Terraform Outputs** — Capture public IP, verify key pair name  
4. **Ansible Deploy** — Install Nginx, deploy static site, configure virtual host  
5. **Health Check** — `curl` public IP until HTTP 200 and content validation

## Prerequisites

### AWS

- AWS account with permissions for EC2, key pairs, and security groups  
- An **existing EC2 Key Pair** in your target region  
- Local or CI AWS credentials (Access Key ID + Secret Access Key)  

### Jenkins

Install plugins:

- Pipeline  
- Credentials Binding  
- SSH Agent  

### Jenkins credentials (required IDs)

| Credential ID | Type | Configuration |
|---------------|------|----------------|
| `aws-credentials-id` | **Username with password** | Username = AWS Access Key ID, Password = Secret Access Key |
| `ssh-key-id` | **SSH Username with private key** | Username = `ubuntu`, Private key = matching EC2 key pair |

### Jenkins job parameters

| Parameter | Description | Example |
|-----------|-------------|---------|
| `AWS_KEY_NAME` | EC2 Key Pair name in AWS | `my-key-pair` |
| `AWS_REGION` | AWS region | `us-east-1` |

### Agent tools

- `git`, `terraform` (≥ 1.5), `ansible-playbook`, `curl`, `ssh`

## Quick start (local Terraform)

```bash
cd terraform
cp terraform.tfvars.example terraform.tfvars
# Edit terraform.tfvars — set aws_key_name and allowed_ssh_cidr

export AWS_ACCESS_KEY_ID="your-access-key"
export AWS_SECRET_ACCESS_KEY="your-secret-key"

terraform init
terraform plan
terraform apply
```

Terraform writes `ansible/inventory/inventory.ini` automatically.

Deploy with Ansible:

```bash
ansible-playbook -i ansible/inventory/inventory.ini ansible/playbook.yml -u ubuntu --private-key ~/.ssh/your-key.pem
```

## Security notes

- Restrict `allowed_ssh_cidr` in `terraform.tfvars` to your IP (`x.x.x.x/32`).  
- Never commit `terraform.tfvars`, private keys, or `inventory.ini` with real IPs to version control.  
- Jenkins uses `StrictHostKeyChecking=no` for CI automation; consider hardening for production.  
- Review security group rules before exposing instances to `0.0.0.0/0`.

## Troubleshooting

| Issue | Check |
|-------|--------|
| Terraform auth failure | Jenkins credential `aws-credentials-id` (username = Access Key) |
| SSH connection refused | Security group port 22, correct `AWS_KEY_NAME`, `ssh-key-id` user = `ubuntu` |
| Ansible host unreachable | `ansible/inventory/inventory.ini` exists after `terraform apply` |
| Health check fails | Nginx running on instance: `sudo systemctl status nginx` |
| Key pair mismatch | `AWS_KEY_NAME` parameter matches EC2 `key_name` output |
